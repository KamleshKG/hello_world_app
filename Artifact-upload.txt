If curl on CMD fails → Chunked Strategy (CMD)
If Step 4 or 5 fails with 413, we split the file right on CMD using PowerShell (built into Windows — no extra tools needed):
Split the file into 900MB chunks:
powershell# Run in PowerShell (not CMD)
$file = "C:\temp\bigfile.tar.gz"
$chunkSize = 900MB
$reader = [System.IO.File]::OpenRead($file)
$buffer = New-Object byte[] $chunkSize
$part = 0
while (($read = $reader.Read($buffer, 0, $buffer.Length)) -gt 0) {
    $outFile = "C:\temp\bigfile.part$part"
    [System.IO.File]::WriteAllBytes($outFile, $buffer[0..($read-1)])
    $part++
}
$reader.Close()
Write-Host "Split into $part chunks"

Artifactory REST API — Three Approaches
Approach A — Simple PUT (Best for large files)
cmdREM Windows CMD - Direct Artifactory PUT (no MyHorizon involved)
curl -X PUT ^
  "https://artifacts-prod1.horizon.bankofamerica.com/artifactory/libs-snapshots-local-generic/com/bofa/hello_flutter/bigfile.tar.gz" ^
  -H "Authorization: Basic BASE64_ARTIFACTORY_CREDS" ^
  -H "X-Checksum-Sha256: YOUR_FILE_SHA256" ^
  -T "C:\temp\bigfile.tar.gz" ^
  -v
Key difference — -T flag streams the file directly instead of -F (multipart). Much better for large files — lower memory, no encoding overhead.

Approach B — Chunked via Artifactory Multipart API
Artifactory has a native multipart upload API designed exactly for large files:
cmdREM Step 1: Initiate multipart upload
